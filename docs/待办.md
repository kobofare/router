# 待办

本文档用于记录当前代码库中已经明确、但尚未完成的工作项。

## 视频能力

### 当前状态

- 已支持把 `veo*` 识别并归属到 `google` 供应商。
- 已支持 `video` 作为系统模型类型，用于供应商管理、渠道模型管理和能力测试展示。
- 已支持在管理后台对渠道执行 `videos` 能力探测。
- 尚未支持通过 Router 对外正式代理视频生成请求。

### 待补充事项

1. 对外代理接口

- 增加公共视频代理入口，例如 `POST /v1/videos`。
- 明确视频请求在 Router 内部的 relay mode、controller、adaptor 走向。
- 统一 OpenAI 风格视频接口与各家兼容实现之间的请求映射规则。

2. 视频计费与用量

- 细化视频模型的计费单位，评估是否仅保留 `per_video`，还是补充 `per_second`、`per_request`、`per_task` 等单位。
- 明确视频任务的计费时机，是创建任务即扣费，还是按结果完成后扣费。
- 定义视频场景下的用量结构，避免继续复用文本 token 语义。

3. 上游协议兼容

- 梳理主流上游的视频接口差异，包括请求路径、认证方式、同步/异步任务模型、状态查询和结果下载。
- 为视频任务补充轮询、取消、失败重试等协议层处理。
- 评估是否需要单独的视频 adaptor，而不是完全复用现有文本/图片链路。

4. 管理后台

- 在供应商模型、渠道模型、分组模型展示中，进一步突出 `video` 能力与视频模型。
- 为视频模型补充更清晰的价格、单位、能力标签展示。
- 在渠道详情页和能力测试结果中，增加更明确的视频测试反馈，例如任务 ID、任务状态、结果地址。

5. 分组与模型开放

- 明确分组面向终端用户开放视频模型时的约束规则。
- 明确视频模型是否需要单独的能力开关，而不是仅按模型名暴露。
- 评估分组层是否需要区分文本、图片、音频、视频四类能力配额。

6. 测试与运维

- 补充视频能力的单元测试与集成测试。
- 增加视频上游联调样例和运维排障说明。
- 明确视频任务失败时的日志字段、trace 透传和可观测性要求。

### 建议优先级

1. 先完成 `POST /v1/videos` 的对外代理闭环。
2. 再补视频计费模型与用量字段。
3. 最后补充后台交互细节、运维文档和更完整的协议兼容。

## OpenAI 定价模型

### 待补充事项

1. 多维定价结构

- 扩展 provider model 的定价结构，支持表达 `search`、`computer-use`、`realtime`、`audio`、`image` 等附加费用，而不是只保留单一的 `input_price` / `output_price`。
- 评估是否需要补充更细粒度的计费维度，例如固定附加费、按请求计费、按时长计费、按图片计费、按分辨率计费。

2. OpenAI 多模态种子补全

- 在定价结构扩展完成后，补充 OpenAI 的 `realtime`、`audio`、`image` 等多模态模型种子。
- 避免在当前单一价格结构下硬编码复杂计费，确保种子数据和真实价格语义一致。

## 复杂计费落库方案

### 待补充事项

1. 扩展价格组件表

- 新增独立的价格组件表，例如 `provider_model_price_components`。
- 允许同一个模型挂多条价格组件，而不是只保留一组 `input_price / output_price`。

2. 价格组件类型

- 首批组件建议支持：
  - `text`
  - `image_generation`
  - `audio_input`
  - `audio_output`
  - `video_generation`
  - `realtime_text`
  - `realtime_audio`

3. 扩展价格单位

- 在现有 `per_1k_tokens / per_1k_chars / per_image / per_video` 之外，补充：
  - `per_second`
  - `per_minute`
  - `per_request`
  - `per_task`

4. 扩展能力表达

- `provider_models` 保留主类型字段用于兼容。
- 新增 `capabilities` 或等价结构，表达一个模型同时具备 `text / image / audio / video` 多能力。

5. 官方来源追溯

- 价格组件级别补充官方来源字段，避免后续复杂计费无法定位来源文档。
