# 表格列宽规范

目标只有三个：

1. 列表优先贴合页面宽度，不先引入水平滚动条。
2. 列宽调整要有统一抓手，不能每个页面各写一套补丁。
3. 操作列按钮尺寸一致，避免不同列表视觉飘忽。

## 1. 适用范围

本规范用于后台常规列表页。

不适用于以下场景：

- 列内容天然不可压缩，且必须完整展示。
- 表格包含大量动态表头或复杂嵌套内容。
- 业务明确要求横向对比，压缩后反而更难读。

这类页面可以保留水平滚动，但要在代码里明确说明原因，不能把 `scroll.x` 当成默认方案。

## 2. 统一实现模式

列表表格统一遵循下面这套模式：

- 表格类名加 `router-table-fit-page`
- 长文本列优先使用 `ellipsis`
- 类型、状态、时间、操作列使用语义类，不直接在页面里散落像素值
- 操作按钮使用统一紧凑样式，不再每个页面单独压高度和宽度

当前通用类：

- `router-table-fit-page`：表格宽度占满页面，`table-layout: fixed`
- `router-table-col-type-narrow`：类型列
- `router-table-col-price-unit`：价格单位列
- `router-table-col-status-compact`：紧凑状态列
- `router-table-col-status-narrow`：常规状态列
- `router-table-col-datetime`：时间列
- `router-table-col-actions-compact`：双按钮操作列
- `router-table-col-actions-icon`：图标型操作列
- `router-table-actions-compact`：双按钮紧凑布局
- `router-table-actions-icon-compact`：单图标紧凑布局

宽度预设统一放在 `web/src/constants/tableWidthPresets.js`。
页面调整列宽时，优先修改这里的预设对象，不要回到页面里散落写数字。

## 3. 列宽分配规则

列宽分配遵循“语义优先”：

- 名称、别名、描述这类长文本列，用百分比或剩余宽度承接
- 类型、状态、币种、价格单位这类短字段，收敛到窄列
- 时间列统一固定宽度
- 操作列统一固定宽度，并反推按钮尺寸

不要再使用“第 3 列 80px、第 4 列 140px”这种位置式口径。应使用语义名，例如 `status`、`actions`、`createdAt`。

## 4. 水平滚动规则

- 常规列表默认不配置 `scroll.x`
- 只有确认存在不可压缩列时，才允许加水平滚动
- 一旦保留水平滚动，代码里需要能看出是哪几列不可压缩

## 5. 调整顺序

后续某个列表宽度需要调整时，按这个顺序处理：

1. 先看是否已使用通用类。
2. 再看是否已接入 `tableWidthPresets` 中的宽度预设。
3. 再看长文本列是否缺少 `ellipsis`。
4. 再看是否误用了 `scroll.x`。
5. 最后才新增页面级特例类。

页面级特例必须说明为什么通用规则不够。

## 6. 当前验证页

这套规范优先在以下页面验证：

- 渠道详情中的模型列表
- 供应商列表
- 套餐列表
- 充值计划列表

后续其他列表继续按同一模式迁移，不再回到逐页打补丁的方式。
