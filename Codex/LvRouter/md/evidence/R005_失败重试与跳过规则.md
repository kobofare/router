# 主张
- R005：仅在未指定渠道且状态码为 429 或 5xx 时重试；每次重试重新选渠并跳过上次失败渠道。

# 证据
- 文件：`internal/admin/controller/relay.go`
  - 函数/符号：`Relay` / `shouldRetry`
  - 行号范围：L70-L97 / L111-L127
  - 片段概述：`shouldRetry` 排除 `SpecificChannelId`；429/5xx 允许重试；重试循环中重新选渠并跳过 `lastFailedChannelId`。

# 结论
- 重试是“基于错误类型 + 重新选渠”的降级策略，而非对同一渠道重复请求。
