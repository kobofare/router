# 主张
- R004：内存缓存关闭时，DB 查询按 `ignoreFirstPriority` 控制：默认仅选最高优先级；忽略优先级时在所有启用渠道中随机。

# 证据
- 文件：`internal/admin/repository/ability/repository.go`
  - 函数/符号：`GetRandomSatisfiedChannel`
  - 行号范围：L27-L54
  - 片段概述：`ignoreFirstPriority=false` 使用 `MAX(priority)` 子查询；为 true 时取消该过滤；并使用 `RANDOM()/RAND()` 随机。

# 结论
- 缓存关闭仍遵循“优先级优先 + 随机分配”的原则。
