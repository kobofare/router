CL

Claude Code

Anthropic 官方 AI 编程助手

环境准备

如果你使用 NPM 安装，需要先安装 Node.js 18 或更高版本；如果使用官方 Native Install，可以跳过这一节。

macOS

# 方法一：Homebrew（推荐）
brew update
brew install node

# 方法二：前往 nodejs.org 下载 LTS 安装包

Windows

# 方法一：从 https://nodejs.org/ 下载 LTS 安装包

# 方法二：Chocolatey
choco install nodejs-lts

# 方法三：Scoop
scoop install nodejs-lts

Linux

# Ubuntu / Debian
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

# CentOS / RHEL / Fedora
curl -fsSL https://rpm.nodesource.com/setup_lts.x | sudo bash -
sudo yum install -y nodejs

安装完成后可验证：

node --version
npm --version

安装

推荐优先使用 Native Install。这种方式没有额外 Node.js 运行时依赖，启动更快，也更适合长期使用。

Native Install（推荐）

macOS

# Homebrew
brew install claude-code

# 或 curl 安装脚本
curl -fsSL https://claude.ai/install.sh | bash

Windows

# PowerShell
irm https://claude.ai/install.ps1 | iex

# CMD
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

Linux

curl -fsSL https://claude.ai/install.sh | bash

# Alpine Linux 额外依赖
apk add libgcc libstdc++ ripgrep
export USE_BUILTIN_RIPGREP=0

NPM

npm install -g @anthropic-ai/claude-code

连接 cch 服务

settings.json（推荐）

在 ~/.claude/settings.json 中写入：

{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "your-api-key-here（你从网站买的key把这一行覆盖掉）",
    "ANTHROPIC_BASE_URL": "https://router.yeying.pub",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1"
  },
  "permissions": {
    "allow": [],
    "deny": []
  }
}

重点：如果输入claude回车后出现：Claude Code can be used with your Claude subscription or billed based on API usage through your Console account。需要让你登录的时候，只需要在.claude同级目录下的.claude.json文件里面加一行 "hasCompletedOnboarding": true，保存后重新打开claude即可。

环境变量（不推荐）

macOS / Linux

# 临时设置
export ANTHROPIC_BASE_URL="https://router.yeying.pub"
export ANTHROPIC_AUTH_TOKEN="your-api-key-here"

# 永久写入 shell 配置
echo 'export ANTHROPIC_BASE_URL="https://router.yeying.pub"' >> ~/.zshrc
echo 'export ANTHROPIC_AUTH_TOKEN="your-api-key-here"' >> ~/.zshrc
source ~/.zshrc

Windows

# 当前会话
$env:ANTHROPIC_BASE_URL = "https://router.yeying.pub"
$env:ANTHROPIC_AUTH_TOKEN = "your-api-key-here"

# 持久化到用户环境变量
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_BASE_URL", "https://router.yeying.pub", [System.EnvironmentVariableTarget]::User)
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_AUTH_TOKEN", "your-api-key-here", [System.EnvironmentVariableTarget]::User)

说明：Anthropic 兼容地址一般 不带 /v1，直接使用站点地址即可。

VS Code 扩展（不推荐使用插件）

  • 在 VS Code 扩展市场搜索并安装 Claude Code
  • 确认 settings.json 或环境变量已经配置完成
  • 如果你接的是 cch 网关，保持上面的自定义地址即可

启动

在项目目录运行：

claude

常见问题

1. 命令未找到

  • 确认全局安装路径已加入系统 PATH
  • 重新打开终端窗口

2. 认证失败

  • 检查 ANTHROPIC_AUTH_TOKEN 是否正确
  • 检查 ANTHROPIC_BASE_URL 是否配置为 https://router.yeying.pub

3. 如何升级

如果你使用安装脚本或包管理器，重新执行安装命令即可升级到最新版本。

codex --version

连接 cch 服务

配置文件方式（推荐）

注意：Windows的配置文件夹在：C:\Users\你的用户名.codex 文件夹（不存在则创建）。 Linux和MAC的在：~/.codex，如果要用访达看，就是在你的用户文件夹里面。 一般安装完输入codex --version之后会自动创建配置文件夹，如果看不到多半是没有打开隐藏文件夹

在 ~/.codex/config.toml 中写入：

model_provider = "cch"
model = "gpt-5.4"
model_reasoning_effort = "xhigh"
disable_response_storage = true
approval_policy = "never"
sandbox_mode = "workspace-write"
personality = "pragmatic"
web_search = "live"
suppress_unstable_features_warning = true

[features]
plan_tool = true
apply_patch_freeform = true
view_image_tool = true
unified_exec = false
streamable_shell = false
rmcp_client = true

[model_providers.cch]
name = "cch"
base_url = "https://router.yeying.pub/v1"
wire_api = "responses"
requires_openai_auth = true

[sandbox_workspace_write]
network_access = true

再创建 ~/.codex/auth.json：

{
  "OPENAI_API_KEY": "your-api-key-here"
}

环境变量方式（不推荐）

如果你更想走环境变量，可以把 requires_openai_auth 改成 env_key：

# ~/.codex/config.toml
model_provider = "cch"
model = "gpt-5.4"
model_reasoning_effort = "xhigh"
disable_response_storage = true
approval_policy = "never"
sandbox_mode = "workspace-write"
personality = "pragmatic"
web_search = "live"
suppress_unstable_features_warning = true

[features]
plan_tool = true
apply_patch_freeform = true
view_image_tool = true
unified_exec = false
streamable_shell = false
rmcp_client = true

[model_providers.cch]
name = "cch"
base_url = "https://router.yeying.pub/v1"
wire_api = "responses"
env_key = "CCH_API_KEY"
requires_openai_auth = true

[sandbox_workspace_write]
network_access = true

然后在终端设置：

export CCH_API_KEY=your-api-key-here

重要：requires_openai_auth 与 env_key 是两种不同认证方式。接 cch 时二选一即可，不要同时写。

VS Code 扩展

  • 安装 Codex – OpenAI's coding agent
  • 然后打开插件，选择使用 apiKey，把激活拿到的key输入进去
  • 打开插件右上角的齿轮设置，选择第一个选项，有一个open config.toml选项，把上面的配置复制进去保存。
  • 然后直接去使用，如果显示401报错，那就重启VS Code再打开重新尝试

GE

Gemini CLI

Google 官方 Gemini 命令行工具

安装

Google 官方当前支持 npx、npm 全局安装和 Homebrew。这里保留最通用的 NPM 方式：

npm install -g @google/gemini-cli

验证安装：

gemini --version

补充：你也可以直接运行 npx @google/gemini-cli，或者在 macOS 上使用 brew install gemini-cli。

连接 cch 服务

配置文件（推荐）

在 ~/.gemini/.env 中配置：

GOOGLE_GEMINI_BASE_URL=https://router.yeying.pub
GOOGLE_API_KEY=your-api-key-here
GEMINI_MODEL=gemini-3.1-pro-preview

创建 settings.json 配置文件 ：

{
  "ide": {
    "enabled": true
  },
  "security": {
    "auth": {
      "selectedType": "gemini-api-key"
    }
  }
}

补充：如果打开gemini请求之后显示apikey无效，还有一个方法，就是直接去你的项目文件夹里面创建.gemini，然后再创建.env把配置配置进去，重启gemini再试即可。

环境变量方式

macOS / Linux

export GOOGLE_GEMINI_BASE_URL="https://router.yeying.pub"
export GOOGLE_API_KEY="your-api-key-here"

Windows PowerShell

$env:GOOGLE_GEMINI_BASE_URL="https://router.yeying.pub"
$env:GOOGLE_API_KEY="your-api-key-here"
$env:GEMINI_MODEL=gemini-3.1-pro-preview

Windows CMD

set GOOGLE_GEMINI_BASE_URL=https://router.yeying.pub
set GOOGLE_API_KEY=your-api-key-here
set GEMINI_MODEL=gemini-3.1-pro-preview

Plan Mode

Gemini CLI 的 Plan / Agent 能力通常依赖你当前工作目录和权限配置。建议：

  • 在项目根目录启动
  • 先确认模型和网关配置已经生效
  • 如果你要访问外部仓库或远端资源，先确认代理和证书环境正确

通用故障排查

安装与配置常见故障排查

安装失败

  • 检查网络连接是否正常
  • 确认有管理员权限（Windows）或使用 sudo（macOS / Linux）
  • 如果是 npm 安装失败，可尝试切换 registry 或代理

API 密钥无效

  • 确认密钥复制时没有多余空格
  • 检查密钥是否已过期或被禁用
  • 确认账号权限是否足够
  • 核对你使用的是正确的兼容接口和模型名称

端点配置错误

  • Claude Code / Droid 的 Anthropic 兼容模型：直接使用站点地址，不带 /v1
  • Codex / OpenAI 兼容模型：使用 https://router.yeying.pub/v1
  • Gemini CLI：使用 https://router.yeying.pub，由 GOOGLE_GEMINI_BASE_URL 提供
  • OpenCode 中 Gemini Provider：使用 https://router.yeying.pub/v1beta

仍然无法解决

建议把以下信息一起记录下来再排查：

  • 终端完整报错
  • 当前使用的工具名称和版本
  • 当前配置文件片段（去掉敏感密钥）
  • 实际请求的网关地址

如果仍然解决不了，再联系客服并附上截图和错误输出。
